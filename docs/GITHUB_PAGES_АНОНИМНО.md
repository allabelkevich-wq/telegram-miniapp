# GitHub Pages: убрать 404 и разместиться анонимно

## Почему 404

Чаще всего 404 на GitHub Pages бывает по двум причинам:

1. **В репозитории не выбран источник «GitHub Actions».**  
   В репозитории код лежит в корне (и есть папка `public/` с `index.html`). Страницы должны собираться из **папки `public`** — это делает уже настроенный workflow. Если в настройках Pages выбран «Deploy from a branch» и папка «root», то в корне репо нет `index.html` → 404.

2. **Неправильный URL.**  
   Для **проектной** страницы (не `username.github.io`) адрес такой:  
   **`https://<USERNAME>.github.io/<REPO_NAME>/`**  
   Например: `https://johndoe.github.io/telegram-miniapp/`  
   Обязательно с именем репозитория в пути и со слэшем в конце для главной.

---

## Что сделать, чтобы 404 пропал

1. Открой репозиторий на GitHub → **Settings** → **Pages** (слева).
2. В блоке **Build and deployment**:
   - **Source:** выбери **GitHub Actions** (не «Deploy from a branch»).
3. Сохрани. Дождись зелёного деплоя (Actions → workflow «Deploy to GitHub Pages»).
4. Открывай сайт по адресу:  
   **`https://<ТВОЙ_USERNAME>.github.io/<ИМЯ_РЕПО>/`**

Если репо называется `telegram-miniapp` и пользователь `myuser`, то:

- Правильно: `https://myuser.github.io/telegram-miniapp/`
- Неправильно: `https://myuser.github.io/` (это сайт пользователя, а не репо).

---

## Размещение «анонимно»

Чтобы не светить основной аккаунт:

1. **Отдельный аккаунт GitHub**
   - Зарегистрируй новый аккаунт (например, почта ProtonMail, SimpleLogin или одноразовая).
   - В профиле не указывай настоящее имя, можно нейтральный ник.

2. **Новый репозиторий**
   - Войди под этим аккаунтом.
   - Создай репозиторий (можно приватный, Pages всё равно будет доступен по ссылке).
   - Залей код (push из своего репо или заново через git).

3. **Включи Pages**
   - Settings → Pages → Source: **GitHub Actions**.
   - Дождись деплоя.

4. **Ссылка для бота**
   - URL приложения будет:  
     **`https://<АНОНИМНЫЙ_USERNAME>.github.io/<REPO>/`**
   - Эту ссылку укажи в BotFather (Menu Button) и в настройках бота как Web App URL.

Так Mini App будет открываться с GitHub Pages, а с твоим основным GitHub-аккаунтом публично связан не будет.

## Кратко

| Проблема | Решение |
|----------|---------|
| 404 при открытии ссылки | Settings → Pages → Source: **GitHub Actions** |
| Не знаю URL | `https://<USER>.github.io/<REPO>/` (со слэшем в конце) |
| Хочу без привязки к основному аккаунту | Отдельный GitHub-аккаунт → новый репо → тот же workflow, другая ссылка в боте |

В репозитории уже есть workflow `.github/workflows/deploy-pages.yml`: при каждом push в `main` он выкладывает содержимое папки **public** на GitHub Pages. Менять код не нужно — только источник в настройках Pages и правильный URL.
