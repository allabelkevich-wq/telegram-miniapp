# Пошаговая инструкция: как выполнить SQL в Supabase

## Что нужно сделать

Создать таблицу `astro_snapshots` и добавить колонки в `track_requests`. Для этого один раз выполняешь готовый SQL в веб-интерфейсе Supabase.

---

## Шаг 1. Открыть файл с SQL

- В Cursor (или в проводнике) открой файл:
  ```
  telegram-miniapp/bot/supabase-migration-astro-and-columns.sql
  ```
- Выдели **весь текст** в файле (Ctrl+A / Cmd+A).
- Скопируй (Ctrl+C / Cmd+C).

---

## Шаг 2. Открыть Supabase в браузере

- Открой браузер и перейди по адресу: **https://supabase.com**
- Войди в свой аккаунт (логин/пароль).
- На главной странице Supabase откроется список проектов. Нажми на **свой проект** (тот, куда сохраняются заявки из бота).

---

## Шаг 3. Открыть SQL Editor

- В левом меню проекта нажми **SQL Editor** (иконка с символом `</>` или пункт «SQL Editor»).
- Справа откроется область для запросов. Нажми кнопку **New query** (или «+ New query»), чтобы создать новый запрос.

---

## Шаг 4. Вставить SQL и запустить

- В большое текстовое поле (где написано что-то вроде «Type your query here») **вставь** скопированный текст (Ctrl+V / Cmd+V).
- Внизу справа нажми зелёную кнопку **Run** (или нажми Ctrl+Enter / Cmd+Enter).
- Подожди пару секунд. Внизу должно появиться сообщение вроде **Success** или **Success. No rows returned** — значит, всё выполнилось.

---

## Шаг 5. Проверить (по желанию)

- В левом меню нажми **Table Editor**.
- В списке таблиц должны быть **track_requests** и **astro_snapshots**. Открой **astro_snapshots** — таблица может быть пустой, это нормально.

---

## Если появилась ошибка

- Скопируй **полный текст ошибки** из красного блока внизу и пришли сюда — подскажу, что исправить.
