# Деплой приложения на Vercel (устарело)

> ⚠️ **Не используется.** Приложение раздаётся с **Render** (`https://telegram-miniapp-ar09.onrender.com/app`). Vercel удалён. Ниже — архивная инструкция на случай, если когда-нибудь понадобится альтернативный хостинг Mini App.

Если репозиторий **приватный**, GitHub Pages на бесплатном плане недоступен. Можно хостить Mini App на **Vercel** — репо может оставаться приватным.

---

## 1. Подключить репозиторий к Vercel

1. Зайди на [vercel.com](https://vercel.com), авторизуйся (через GitHub).
2. **Add New** → **Project** → выбери репозиторий `telegram-miniapp` (или как он у тебя назван).
3. В настройках проекта:
   - **Root Directory:** нажми **Edit**, укажи **`public`** и сохрани.  
     Так в корне сайта будут раздаваться файлы из папки `public/` (в том числе `index.html`).
   - **Build Command** можно оставить пустым (статический сайт).
   - **Output Directory** — по умолчанию не трогай (корень уже будет `public`).
4. Нажми **Deploy**. Дождись окончания сборки.

После деплоя будет URL вида: `https://telegram-miniapp-xxx.vercel.app` (или твой домен).

---

## 2. Указать URL приложения в боте

В `bot/.env` задай переменную **MINI_APP_URL** — полный URL приложения на Vercel (без параметров в конце, слеш в конце можно):

```env
MINI_APP_URL=https://telegram-miniapp-six-teal.vercel.app
```

В коде бота к этому URL автоматически добавляется `?v=6` (или текущая версия) для сброса кэша.

Перезапусти бота (`npm start` в папке `bot`). В логе должно быть:  
`Кнопка меню установлена: https://telegram-miniapp-six-teal.vercel.app?v=6`

---

## 3. Дальнейшие обновления

При пуше в ветку `main` Vercel по умолчанию делает новый деплой. После деплоя пользователям достаточно обновить приложение в Telegram (закрыть и открыть снова), чтобы подтянуть новую версию.

---

## Итог

| Где        | Что сделать |
|-----------|-------------|
| Vercel    | Подключить репо, **Root Directory** = `public`, задеплоить. |
| bot/.env  | Прописать `MINI_APP_URL=https://telegram-miniapp-six-teal.vercel.app` |
| Бот       | Перезапустить после смены `.env`. |
